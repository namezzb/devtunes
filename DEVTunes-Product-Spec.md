## 1. 产品定位

### 1.1 一句话描述

为独立开发者和互联网从业者打造的私人音乐Agent——通过导入网易云歌单播放 + Claude Code风格的AI对话，在开发工作中提供沉浸式音乐体验和即时AI辅助。

### 1.2 目标用户

| 用户画像 | 特点 |
|----------|------|
| **独立开发者** | 长时间coding，需要音乐陪伴+AI辅助 |
| **产品经理** | 需要快速获取信息、整理思路 |
| **运营从业者** | 多任务处理，需要高效工具 |

### 1.3 核心价值

- 开发工作 + 音乐 + AI 三位一体
- 减少工作疲劳，提升专注力
- 隐私优先，对话历史本地存储

## 3. 音乐播放技术方案

### 3.1 核心技术选型

| 组件 | 选择 | 说明 |
|------|------|------|
| **音乐API** | NeteaseCloudMusicApi | 开源项目，逆向网易云API |
| **项目地址** | https://github.com/Binaryify/NeteaseCloudMusicApi | 30k+ Star |
| **部署方式** | 自建服务器 | 部署在我们自己的Node.js后端 |
| **音频播放** | 浏览器 audio 标签 + Web Audio API | 音频URL来自网易云CDN |

### 3.2 播放原理

```
用户操作流程：
1. 用户粘贴歌单链接
   例如：https://music.163.com/playlist?id=123456789

2. 我们的后端调用 NeteaseCloudMusicApi：
   GET /playlist/track/all?id=123456789

3. API 返回：
   - 歌单名称、封面、描述
   - 歌曲列表（歌名、艺术家、专辑封面）
   - 每首歌的播放URL：
     {
       name: "晴天",
                       artist: "周杰伦",
       url: "https://m10.music.126.net/xxx.mp3"
     }

4. 前端用 <audio src="https://m10.music.126.net/xxx.mp3"> 播放
   - 浏览器从网易云CDN边下载边播放
   - 无需下载到本地

5. Web Audio API 捕获音频数据实现可视化
```

### 3.3 后端API接口

```javascript
// 歌单相关
GET  /api/playlist/:id           // 获取歌单信息
GET  /api/playlist/track/all    // 获取歌单所有歌曲+播放URL

// 搜索相关
GET  /api/search                 // 搜索歌曲

// 歌曲相关
GET  /api/song/url               // 获取单曲播放URL
GET  /api/lyric                  // 获取歌词
```

### 3.4 数据结构

```typescript
// 歌单
interface Playlist {
  id: number;
  name: string;
  coverImgUrl: string;
  description: string;
  trackCount: number;
  playCount: number;
  creator: {
    nickname: string;
    avatarUrl: string;
  };
}

// 歌曲
interface Track {
  id: number;
  name: string;              // 歌曲名
  artists: Artist[];          // 艺术家
  album: {
    name: string;            // 专辑名
    picUrl: string;          // 封面
  };
  duration: number;          // 时长(ms)
  url?: string;              // 播放URL（需要单独获取）
}

// 艺术家
interface Artist {
  id: number;
  name: string;
}
```

### 3.5 已知问题与应对

| 问题 | 发生概率 | 应对方案 |
|------|----------|----------|
| 歌曲无版权无法播放 | 10-20% | 显示"该歌曲暂时无法播放"，自动跳下一首 |
| 播放URL获取失败 | 5% | 重试机制，最多3次 |
| 歌单导入失败 | 5% | 检查链接格式，提示正确格式 |
| 播放URL过期 | 偶尔 | 播放前检测，失效则重新获取 |

**成功率**: 约80-90%的歌曲可正常播放

### 5.4 特效详细规格

#### 黑洞效果 (Black Hole)

```
- 位置：右下角悬浮，固定定位
- 外观：中心黑色圆形(100px)，周围光晕扭曲
- 动画：持续的缓慢旋转（360° / 30s，linear）
- 交互：hover时光晕扩大，播放音乐时脉冲(scale 1.0 → 1.05)
- 层级：z-index: 1，在星空粒子之下
```

#### 极光 (Aurora)

```
- 位置：顶部横条，高度150px
- 颜色：透明渐变，青绿(0%) → 紫蓝(50%) → 粉红(100%)
- 动画：8-12s循环，noise驱动有机流动
- 交互：播放时opacity 0.8，暂停时opacity 0.3
- 模糊：下边缘模糊过渡
```

#### 星空粒子 (Star Field)

```
- 数量：300-500个
- 大小：1-3px，随机
- 亮度：0.3-1.0，随机
- 颜色：90%白色，10%淡蓝(#87CEEB)或淡黄(#FFE4B5)
- 动画：缓慢漂浮，视差效果跟随鼠标
- 闪烁：随机5%的星星有闪烁动画
```

#### 波形可视化 (Audio Visualizer)

```
- 位置：播放器中央
- 模式：圆形频谱（默认）/ 柱状频谱 / 波形线
- 颜色：跟随极光渐变（青→紫→粉）
- 动画：实时响应音频频率数据
- 暂停：保持最后帧，opacity降低
```

## 附录：歌单导入格式

### 支持的链接格式

```
https://music.163.com/playlist?id=123456789          # 网页版
https://y.music.163.com/m/playlist?id=123456789    # 移动版
```

### 提取ID方式

```javascript
// 从URL提取歌单ID
function extractPlaylistId(url) {
  const match = url.match(/id=(\d+)/);
  return match ? match[1] : null;
}
```
